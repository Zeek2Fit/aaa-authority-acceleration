# Trust Signal Module Templates

## MODULE 1: CASE STUDIES
**E-E-A-T: EXPERIENCE**

### Purpose
Demonstrate real-world results and client transformations.

### Generation Process
1. Extract transformation journey from Brand Profile (before/after)
2. Use competitor structures that work for case studies
3. Apply user's voice patterns
4. Create templates fillable with real client data

### Templates

**Template 1: Transformation Story**
Structure: Problem → Struggle → Discovery → Solution → Results
- Opening Hook (based on competitor hooks + voice)
- The Problem (Before State from Brand Profile)
- The Journey (methodology steps)
- The Results (transformation with metric placeholders)
- Key Takeaway (reinforces positioning)

**Template 2: Quick Win Case Study**
Structure: Challenge → Single Action → Result

**Template 3: Comparison Case Study**
Structure: Client A (your method) vs Client B (old method)

### Fill-In Prompts
- Client name/anonymous identifier
- Specific before metrics
- Timeline
- Specific after metrics
- Quote from client (optional)

## MODULE 2: FRAMEWORKS
**E-E-A-T: EXPERTISE**

### Purpose
Create proprietary methodologies demonstrating deep expertise.

### Framework Types
- **Acronym Framework** - Each letter = step (e.g., S.M.A.R.T.)
- **Numbered System** - X steps/pillars (e.g., 7 Habits)
- **Matrix/Quadrant** - 2x2 categorization
- **Ladder/Pyramid** - Hierarchical progression
- **Cycle/Wheel** - Iterative process

### Template Structure
- Framework Name + Type
- Gap it fills (from competitor analysis)
- Visual representation
- Component breakdown with application
- Why this works (unique insight)
- Content pieces derivable
